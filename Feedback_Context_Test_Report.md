# 上下文 / 反馈对 BOQ 生成的影响测试报告
# Context & Feedback Impact Test — Case 775368 (New Habshan 400/220kV)

> 输入：`N-20546 - Habshan SS SoW_revG - Lot 1.pdf` + `N-20546 EA 7000 P 103_D - SLD (Recovered 1).pdf`
> 对比基准：业务标注的真实 BOQ `BOQ-WEB-48DB5A88 - 775368.xlsx`（Actual BOQ sheet）
> 测试三种情形：① 基线（无 Step1 上下文、无 Step2 反馈，= `BOQ-WEB-AFB7B80D.xlsx`）② 加正确 Step1 上下文 ③ 对错误产品加 Step2 反馈

---

## 一、结论速览

- **Step1 上下文** 的主要作用是**剔除超范围的产品族**（GIS 局放、变压器测温/辅助传感器）——与业务批注一致；但**不改数量、也不改产品型号/结构**（数量仍是 39/13/39，量级仍错）。
- **Step2 反馈** 效果最强：**删除了全部超范围/错误行，并把数量修正到业务期望值**（PMU 40→14、iQ+ 14→1），与批注完全对齐。局限：无法把某行替换成"不在该场景候选目录里"的产品（PQ 想换成 IDM+ 失败 → 被直接删除），也不会自动生成盘级结构。
- **两者都提升了准确度，但都没能达到真实 BOQ 的"工程化盘级配置"结构**——印证了差距分析：当前工具是"产品识别器"，上下文/反馈能纠偏，但要变成"报价配置器"仍需 P0 的配置模板工作。
- 测试中**发现并修复了一个真实 bug**：反馈动作为 `replace` 时修正数量未被应用（已在 `webapp/server.py` 修复，`replace` 与 `adjust` 都会应用数量）。
- 另发现 **LLM 偶发不稳定**：一次重算返回空结果（需加重试）。

## 三、逐情形分析

### ①基线（无上下文/反馈）—— = `BOQ-WEB-AFB7B80D.xlsx`
8 行扁平品类汇总，含 4 个超范围/错误项（QTMS 软件、QGateway、GIS 局放、变压器测温）和 3 个数量严重偏大的项（PMU 40、iQ+ 14、PQ 40）。与之前差距分析结论一致。

### ②加正确 Step1 上下文
上下文明确写明"仅 FMS/PQM/PMU 属 Qualitrol 范围；GIS 局放/变压器测温不在范围；数量按监测间隔"。
- **正确剔除**了 GIS 局放、变压器测温、变压器辅助传感 3 个产品族（从 8 行降到 5 行）✓
- 但 **软件/网关仍被保留**，**数量几乎没变**（40→39、14→13、40→39，量级依旧错）。
- 原因：上下文是注入到 **LLM 的场景/需求判断**，能影响"是否在范围"，但**产品数量是规则按间隔/回路计数得出**，上下文没有改这条规则链。

### ③加 Step2 反馈（对错误产品逐条 👎 + 评论）
LLM 依据每条评论决策：
- QTMS 软件、QGateway、GIS 局放、变压器测温 → **remove**（超范围）✓
- HATHAWAY IDM（PMU）→ **adjust 40→14** ✓
- iQ+ → **adjust 14→1** ✓
- BEN 6000（PQ）→ **remove**（应转成 IDM+×9，但 IDM+ 不在该场景候选目录里，LLM 只能删）
- 结果：从 8 行精简为 3 行（PMU×14、iQ+×1、iSGM×0），**数量与业务期望一致**。

## 四、发现的问题与改进项

1. **[已修复] replace 未带数量**：反馈动作为 `replace` 时若同时给了修正数量，之前未应用；现已改为 `replace`/`adjust` 都应用数量（`webapp/server.py`、测试脚本同步修复）。
2. **[待做｜数据包] 候选目录太窄**：真实项目里 FMS/PQM/PMU 都用 **IDM+**，但数据包中 IDM+ 未作为 PMU/PQ 场景的候选产品，导致"把 PQ 换成 IDM+"无法实现、只能删行。建议在数据包把 IDM+/Informa 补充为这些场景的候选。
3. **[待做｜稳健性] LLM 偶发返回空**：一次重算 `regenerate_boq_lines` 返回 0 决策（当前会提示"未返回可用修订"）。建议加 1–2 次自动重试。
4. **[结构性｜P0] 仍是扁平清单**：即便反馈修正后，输出仍是"品类×数量"，不是盘级工程配置。要贴近真实 BOQ，仍需按差距分析的 P0——用真实 BOQ 沉淀"按间隔配 DAU + 配件 + 盘柜 + 服务"的配置模板。

---

## 五、总体判断

- **上下文**：擅长"收窄范围"（去掉不该报的产品族），适合在分析前就把 scope 讲清楚。
- **反馈**：擅长"精确纠偏"（删错误行、改数量到位），适合业务看到结果后逐条修正，且**数据留痕可用于后续优化**。
- 两者叠加能把基线的 8 行错误清单，纠正到"范围正确 + 关键数量正确"的 3 行——**方向性成功**；但要达到真实 BOQ 的工程化结构，仍需配置模板层（P0）与数据包候选补全。
